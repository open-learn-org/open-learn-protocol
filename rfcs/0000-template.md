# RFC NNNN — Title

## Summary

One paragraph. What does this change? Read this and decide if you care.

## Motivation

Why does the spec need this change? What does it enable that wasn't possible before, or what problem does it fix that's biting implementers today?

Include evidence: links to issues, examples from real integrations, citations from other specs. "It would be nice to have" is not a motivation.

## Detailed design

The change in full. If it's an addition to a spec, write it as if it were going into that spec — same level of detail and prescriptiveness (MUST / SHOULD / MAY).

If it touches multiple files, list each file and what changes in it.

If it's a new spec, draft the `protocol.md` here. The implementer specs come after acceptance.

## Drawbacks

Why might we *not* do this? Be honest. If you can't list any, you haven't thought hard enough.

## Alternatives considered

What other shapes did you think about? Why is this one better? "I didn't consider any" is a red flag.

## Unresolved questions

What's still open? What would you like reviewers to focus on?

## Adoption

If accepted, who has to change what? Reporters? Collectors? Both? Is there a transition period?
